# Why DragonClaw?

## The Problem

OpenClaw is brilliant, but it was built for Western users. The integrations are WhatsApp, iMessage, Gmail, Spotify. The default LLMs are Claude and GPT. The community skills assume English-language services.

If you're a Chinese user, you face several problems:

1. **Wrong chat platforms.** You use WeChat, DingTalk, and Feishu — not WhatsApp and iMessage. OpenClaw has no built-in support for these.

2. **Expensive LLMs.** Claude and GPT-4o cost 10-20x more than DeepSeek or Qwen for equivalent quality on Chinese-language tasks.

3. **No crypto built-in.** You have to find, install, and configure Binance skills separately. Most Chinese crypto users want this from day one.

4. **English-first documentation.** The setup guides, skill descriptions, and error messages all assume English.

## The Solution

DragonClaw solves all of these:

| Problem | OpenClaw | DragonClaw |
|---------|----------|------------|
| Chat apps | WhatsApp, Telegram, iMessage | WeChat, DingTalk, Feishu + Telegram, Discord |
| Default LLM | Claude / GPT | DeepSeek / Qwen / Kimi / GLM |
| Crypto trading | Install separately | 7 Binance skills hardcoded |
| Language | English-first | Chinese-first, bilingual |
| Cost per query | ~$0.01-0.03 (Claude) | ~$0.001-0.003 (DeepSeek) |

## Who Is It For?

- **Chinese crypto traders** who want an AI assistant that can trade on Binance, audit contracts, and track memes from a DingTalk or WeChat message
- **Developers** building AI agents for the Chinese market who need a ready-made framework with Chinese platform connectors
- **Teams** using Feishu or DingTalk who want an always-on AI colleague that remembers context and executes tasks autonomously
- **Anyone** who wants OpenClaw's power with Chinese ecosystem support and much lower running costs
